## PHASE 3 ‚Äî State Persistence & Recovery Tests (CRITICAL)

### 3a ‚Äî Normal shutdown
- [x] Generate 100 blocks with transactions
- [x] Graceful shutdown (Ctrl+C / SIGTERM)
- [x] Restart node
- [x] Block number is exactly where it stopped (verified: block 12 ‚Üí restored block 12)
- [ ] All transactions are in LevelDB
- [x] All account balances restored correctly (verified: 10001 ETH after 1 ETH tx restored)
- [ ] All contract state restored correctly

### 3b ‚Äî Sudden crash recovery (HIGH PRIORITY)
- [ ] Generate 1,000 blocks rapidly
- [ ] SUDDEN kill: `kill -9 <pid>` (SIGKILL ‚Äî no graceful shutdown)
- [ ] Restart node
- [ ] LevelDB integrity check passes (no corruption)
- [ ] Block number is close to where it stopped (within last checkpoint)
- [ ] State is consistent (no half-written blocks)
- [ ] No panic/crash on restart after SIGKILL

### 3c ‚Äî State checkpointing
- [ ] State written to LevelDB every N blocks (configurable interval)
- [ ] `--state-interval 5` checkpoints every 5 seconds
- [ ] After SIGKILL, loss is at most 1 checkpoint interval
- [ ] Checkpoint log shows timestamps

### 3d ‚Äî DB integrity
- [ ] LevelDB opens without error after crash
- [ ] Sub-databases (blocks, txs, receipts) are consistent with each other
- [ ] Block hash chain is valid (each block's parentHash matches previous)
- [ ] No orphan transactions in tx DB
- [ ] Storage trie is consistent
